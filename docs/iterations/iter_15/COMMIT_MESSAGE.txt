add loongarch busybox non-scoring stall diagnostics

Add a cfg-gated LoongArch BusyBox diagnostic mode for investigating
commands that can stall or fault before clean exit. The diagnostic runner
executes one real /musl/busybox command at a time without emitting official
testcase success/fail lines, then reports entry PC, last observed ERA/BADV,
ECODE, syscall count, trap count, same-ERA count, timer ticks, and outcome.

Add diagnostic state ownership to the LoongArch user run-state module and a
trap observation/timer hook in the LoongArch trap module. Add a small
LOONGARCH_RUSTFLAGS Makefile hook so local diagnostics can build a separate
kernel-la-diag with --cfg loongarch64_busybox_diag while normal make all and
official scoring builds remain unchanged.

- basename /aaa/bbb exits with code 0 in the isolated diagnostic path.
- printf "abc\n" exits with code 0 in the isolated diagnostic path.
- uname exits with code 0 in the isolated diagnostic path.
