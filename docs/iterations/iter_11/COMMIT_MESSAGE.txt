fix loongarch waitpid status encoding for full basic-musl score

Close the LoongArch basic-musl scoring gap by fixing wait status writeback in
the LoongArch process subsystem. Child exits now record only the low 8-bit
exit code, and wait4 writes Linux-compatible normal-exit status
((exit_code & 0xff) << 8) through the existing user-copy path. This lets the
real /musl/basic/waitpid program observe WEXITSTATUS(status) == 3 and print
the expected success output.

Add a narrow LoongArch user-entry synchronization path for the fixed-address
BusyBox ET_EXEC runner. The now-successful waitpid path exposed timing
sensitivity when BusyBox command tracing was quiet; user_mmu.rs now provides a
silent ibar/dbar settle loop, real_elf.rs calls it after activating fixed user
mappings, and trap.rs calls it before returning to PLV3.

Keep runtime_dispatch.rs unchanged and preserve the existing BusyBox-la safe
command set.
