the parent syscall handler is still live on the trap stack. Each trap reserves
a 16 KiB stack slice and records the previous slice top in the trap frame so
return restores the cursor without corrupting parent kernel state.

fork-style clone execution, child exit recording, wait4 status writeback,
user memory snapshots, fd/cwd snapshots, and safe user-copy helpers.

- exit
- wait
- waitpid
- yield
- fork

runtime_dispatch.rs.
